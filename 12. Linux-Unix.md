# Linux/Unix Interview Questions & Answers for Beginners

A concise reference of 10 essential Linux/Unix interview questions with clear answers and example commands.

---

## 1. What is the difference between Linux and Unix?

**Answer:**  
Unix is a proprietary operating system developed in the 1960s at Bell Labs. Linux is a free, open-source OS created by Linus Torvalds in 1991, inspired by Unix but written from scratch. Linux follows Unix principles and is Unix-like, but they are not the same system.

---

## 2. What does the command `ls` do, and how do you show hidden files?

**Answer:**  
The `ls` command lists the contents of a directory. To also show hidden files (files starting with a dot `.`), use the `-a` flag:

```bash
ls -a
```

---

## 3. How do you navigate between directories in the terminal?

**Answer:**  
Use the `cd` (change directory) command. Common uses:

```bash
cd /home/user      # Go to an absolute path
cd Documents       # Go into a subfolder
cd ..              # Go up one level
cd ~               # Go to your home directory
```

---

## 4. What is the difference between absolute and relative paths?

**Answer:**  
- **Absolute path** — starts from the root `/` and gives the full location of a file.  
- **Relative path** — is relative to your current working directory and does not start with `/`.

```bash
/home/user/docs/file.txt   # Absolute path
docs/file.txt              # Relative path
```

---

## 5. How do you view the contents of a file without opening an editor?

**Answer:**  
Several commands can be used:

| Command | Description |
|---------|-------------|
| `cat`   | Prints the entire file at once |
| `less`  | Lets you scroll through a file page by page |
| `head`  | Shows the first 10 lines |
| `tail`  | Shows the last 10 lines |

```bash
cat file.txt
less file.txt
head file.txt
tail file.txt
```

---

## 6. What does `chmod` do, and what is `chmod 755`?

**Answer:**  
The `chmod` command changes file permissions. Permissions are set for three groups: **owner**, **group**, and **others**. Each digit is the sum of:

| Value | Permission |
|-------|------------|
| 4     | Read (r)   |
| 2     | Write (w)  |
| 1     | Execute (x)|

So `755` means:
- **7** (owner): read + write + execute
- **5** (group): read + execute
- **5** (others): read + execute

```bash
chmod 755 script.sh
```

## 7. What is a superuser (root) and how do you run a command as root?

**Answer:**  
The **root** user is the system administrator with full access to all commands and files. Use it with caution. To run a single command with root privileges, prepend `sudo` (superuser do):

```bash
sudo apt update
```

---

## 8. How do you search for text inside a file?

**Answer:**  
Use the `grep` command followed by the pattern and filename:

```bash
grep 'error' logfile.txt           # Basic search
grep -i 'warning' logfile.txt      # Case-insensitive search
grep -r 'TODO' ./project/          # Recursive search in a directory
```

---

## 9. What is a process, and how do you list running processes?

**Answer:**  
A **process** is a running instance of a program. Use `ps` to list processes in the current session, or `ps aux` for all system processes. Use `top` or `htop` for a live, interactive view.

```bash
ps aux    # List all running processes
top       # Live view of processes
```

---

## 10. How do you redirect output to a file, and what is piping?

**Answer:**  
- `>` redirects output to a file (overwrites).  
- `>>` appends output to a file.  
- `|` (pipe) sends the output of one command as input to another.

```bash
ls > filelist.txt          # Save output to a file (overwrite)
ls >> filelist.txt         # Append output to a file
ps aux | grep nginx        # Filter process list for 'nginx'
```
