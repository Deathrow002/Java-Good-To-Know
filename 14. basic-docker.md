# Basic Docker Interview Questions & Answers

This document contains 10 essential interview questions and detailed answers focusing on Basic Docker concepts. It is designed for entry-level developers, DevOps beginners, or anyone preparing for a technical interview on containerization.

---

### **1. What is Docker, and why is it used?**
**Answer:** Docker is an open-source platform that allows developers to automate the deployment, scaling, and management of applications using containerization. It packages an application and its necessary dependencies (libraries, system tools, code) into a single unit called a container. This ensures that the application runs consistently regardless of the environment (e.g., a developer's local laptop, a testing server, or a production environment), effectively eliminating the "it works on my machine" problem.

---

### **2. What is the difference between a Docker image and a Docker container?**
**Answer:** * **Docker Image:** A read-only template or blueprint used to create containers. It contains the application code, runtime, libraries, and environment variables needed to run the software. Images are built layer by layer and are immutable (cannot be changed once created).
* **Docker Container:** A running, isolated instance of a Docker image. It is the dynamic environment where the application actually executes. You can run multiple containers simultaneously from a single image.

---

### **3. How does a Docker container differ from a Virtual Machine (VM)?**
**Answer:** * **Containers:** Share the host system's operating system (OS) kernel and isolate applications at the process level. This makes them extremely lightweight, fast to start (usually in seconds), and highly resource-efficient.
* **Virtual Machines:** Require a hypervisor and run a completely separate guest OS for every instance. This makes VMs much heavier, slower to boot (minutes), and more resource-hungry compared to containers.

---

### **4. What is a Dockerfile?**
**Answer:** A `Dockerfile` is a plain-text configuration script containing a sequential series of instructions and commands used to build a Docker image automatically. Each command in the file (such as `FROM`, `RUN`, `COPY`, `CMD`) creates a new layer in the resulting image, enabling version control and reproducibility for application environments.

---

### **5. What is the difference between `docker run` and `docker start`?**
**Answer:** * **`docker run`**: This command creates a *brand new* container from a specified image and then starts it. It is essentially a combination of `docker create` and `docker start`.
* **`docker start`**: This command restarts an *existing, stopped* container. It does not create a new container instance and retains whatever state or configuration was left in that specific container.

---

### **6. Explain the basic components of Docker Architecture.**
**Answer:** Docker uses a client-server architecture consisting of three primary components:
1. **Docker Client:** The primary interface (like the CLI) where users type commands (e.g., `docker build`, `docker run`). It sends these commands to the Docker Daemon.
2. **Docker Host (Daemon):** The background service (`dockerd`) running on the host machine. It listens for client requests and does the heavy lifting of building, running, and managing containers and images.
3. **Docker Registry:** A centralized storage location for Docker images (e.g., Docker Hub, AWS Elastic Container Registry). The daemon pulls images from or pushes images to the registry.

---

### **7. What are Docker Volumes, and why are they necessary?**
**Answer:** By default, data created inside a container is ephemeral—it is lost when the container stops or is deleted. **Docker Volumes** are mechanisms used to persist data generated by and used by Docker containers. They map a directory on the host machine to a directory inside the container, ensuring that databases, user uploads, logs, and other critical data survive the container's lifecycle.

---

### **8. What is the difference between `COPY` and `ADD` in a Dockerfile?**
**Answer:** Both commands are used to bring files from the host machine into the Docker image, but they have distinct functionalities:
* **`COPY`**: Only copies local files and directories from the host into the image. It is straightforward and generally preferred for standard file transfers.
* **`ADD`**: Can do what `COPY` does, but has extra features: it can download files from remote URLs and automatically extract compressed archives (like `.tar.gz` files) directly into the target directory inside the image.

---

### **9. What is Docker Hub?**
**Answer:** Docker Hub is the world’s largest public, cloud-based repository (registry) for Docker images maintained by Docker Inc. It serves as a central marketplace where developers can find officially maintained base images (such as Node.js, Python, Ubuntu, or Nginx), share their custom images publicly or privately, and automate build workflows.

---

### **10. How do you remove an image, and what must you do before removing it?**
**Answer:** You remove an image using the `docker rmi <image_id>` or `docker image rm <image_name>` command. However, you cannot delete an image if it is currently being used by any container (even a stopped one). Therefore, the strict prerequisite is to first stop and remove all containers referencing that specific image using `docker stop <container_id>` followed by `docker rm <container_id>`.
