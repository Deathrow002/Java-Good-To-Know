## Core Concepts

### Q1. What is a shell script and why is it used?

**Answer:**

A **shell script** is a plain text file containing a sequence of commands that the shell (command-line interpreter) executes one by one. It automates tasks you would otherwise type manually in the terminal.

**Common uses:**
- Automating repetitive tasks (backups, deployments, log rotation)
- System administration (user management, disk monitoring)
- CI/CD pipelines (build, test, deploy steps)
- File processing and batch operations
- Scheduling jobs with cron

**Simple example:**
```bash
#!/bin/bash
echo "Hello, World!"
date
whoami
```

Running it:
```bash
chmod +x hello.sh   # make it executable
./hello.sh          # run it
```

### Q2. What is a shebang (`#!`) and why is it important?

**Answer:**

The **shebang** (also called hashbang) is the very first line of a script. It tells the operating system **which interpreter** to use to execute the file.

```bash
#!/bin/bash       # use Bash shell
#!/bin/sh         # use POSIX sh (more portable)
#!/usr/bin/env python3   # use Python 3
#!/usr/bin/env node      # use Node.js
```

**Why it matters:**
- Without a shebang, the OS may not know how to run the file
- It ensures the correct interpreter is used regardless of the user's default shell
- `#!/usr/bin/env bash` is preferred over `#!/bin/bash` because it finds `bash` wherever it's installed on the system

```bash
# With shebang — can run directly
./myscript.sh

# Without shebang — must specify interpreter manually
bash myscript.sh
```

### Q3. What is the difference between `sh`, `bash`, `zsh`, and `dash`?

**Answer:**

| Shell | Full Name | Notes |
|---|---|---|
| `sh` | Bourne Shell | Original UNIX shell; POSIX standard |
| `bash` | Bourne Again Shell | Most common Linux default; superset of `sh` |
| `zsh` | Z Shell | macOS default since Catalina; feature-rich |
| `dash` | Debian Almquist Shell | Lightweight; Ubuntu's `/bin/sh` default |
| `fish` | Friendly Interactive Shell | User-friendly; not POSIX-compatible |

**Key differences:**
- `bash` has features `sh` doesn't: arrays, `[[ ]]`, `(( ))`, string manipulation
- Scripts using `#!/bin/sh` should only use POSIX features for maximum portability
- `dash` is faster than `bash` for scripts (used in Ubuntu boot scripts)
- `zsh` is mostly `bash`-compatible but has additional features

**Rule of thumb:** Write scripts with `#!/bin/bash` unless you need maximum portability across all Unix systems, in which case use `#!/bin/sh` and stick to POSIX syntax.

### Q4. How do you make a script executable and run it?

**Answer:**

```bash
# Step 1: Create the script
cat > myscript.sh << 'EOF'
#!/bin/bash
echo "Script is running"
EOF

# Step 2: Make it executable
chmod +x myscript.sh

# Step 3: Run it
./myscript.sh              # run from current directory
bash myscript.sh           # run explicitly with bash
sh myscript.sh             # run with sh
source myscript.sh         # run in current shell (shares variables)
. myscript.sh              # same as source
```

**chmod permission bits:**
```bash
chmod +x file       # add execute for all users
chmod 755 file      # rwxr-xr-x (owner: rwx, group: rx, others: rx)
chmod 700 file      # rwx------ (only owner can read/write/execute)
```

**Difference between `./script.sh` and `source script.sh`:**
- `./script.sh` — runs in a **new subshell**; variables/changes don't affect the current shell
- `source script.sh` — runs in the **current shell**; changes (variables, `cd`) persist

### Q5. What are exit codes and how do you use them?

**Answer:**

Every command in Linux returns an **exit code** (also called return code) when it finishes:
- `0` — **success**
- Non-zero (1–255) — **failure** or error

```bash
# Check exit code of last command
ls /tmp
echo $?      # 0 (success)

ls /nonexistent
echo $?      # 2 (no such file or directory)

# Set exit code in your script
exit 0       # success
exit 1       # generic error
exit 2       # misuse of shell command

# Use exit codes in logic
if grep -q "error" logfile.txt; then
    echo "Errors found"
    exit 1
fi
echo "All good"
exit 0
```

**Common exit codes:**

| Code | Meaning |
|---|---|
| 0 | Success |
| 1 | General error |
| 2 | Misuse of shell built-ins |
| 126 | Command found but not executable |
| 127 | Command not found |
| 128+N | Fatal signal N received |
| 130 | Script terminated by Ctrl+C (SIGINT) |

## Variables & Data Types

### Q6. How do you declare and use variables in bash?

**Answer:**

```bash
#!/bin/bash

# Declare a variable (NO spaces around =)
name="Alice"
age=30
pi=3.14

# Use a variable with $
echo "Name: $name"
echo "Age: $age"

# Curly braces for clarity (recommended)
echo "Name: ${name}"
echo "File: ${name}_backup.tar.gz"

# Command substitution — store command output in variable
today=$(date +%Y-%m-%d)
files=$(ls /tmp)
count=$(wc -l < file.txt)

echo "Today is: $today"

# Read-only variable (constant)
readonly MAX_RETRIES=3

# Unset (delete) a variable
unset name
```

**Important rules:**
- No spaces around `=` — `name = "Alice"` is a **syntax error**
- Variable names are case-sensitive: `Name` ≠ `name`
- No data types — everything is a string by default
- Use `$(...)` for command substitution (preferred over backticks `` ` `` )

### Q7. What are special variables in bash?

**Answer:**

Bash provides built-in special variables:

| Variable | Meaning |
|---|---|
| `$0` | Name of the script |
| `$1`, `$2`, ... | Positional arguments (1st, 2nd, ...) |
| `$#` | Number of arguments passed |
| `$@` | All arguments as separate words |
| `$*` | All arguments as a single string |
| `$$` | PID (Process ID) of current shell |
| `$!` | PID of last background process |
| `$?` | Exit code of last command |
| `$_` | Last argument of previous command |

```bash
#!/bin/bash
# Save as demo.sh and run: ./demo.sh hello world 42

echo "Script name:     $0"    # ./demo.sh
echo "First arg:       $1"    # hello
echo "Second arg:      $2"    # world
echo "All args:        $@"    # hello world 42
echo "Argument count:  $#"    # 3
echo "Current PID:     $$"

ls /tmp
echo "Exit code of ls: $?"    # 0
```

### Q8. What is the difference between `$@` and `$*`?

**Answer:**

Both hold all positional arguments, but they behave differently **when quoted**:

| | Unquoted | Quoted |
|---|---|---|
| `$*` | Same as `$@` | All args as **one string** |
| `$@` | Same as `$*` | Each arg as a **separate word** |

```bash
#!/bin/bash
# Run as: ./test.sh "hello world" foo

print_args() {
    echo "Number of args received: $#"
}

# With "$@" — preserves word boundaries (CORRECT)
print_args "$@"      # receives 2 args: "hello world" and "foo"

# With "$*" — joins everything (usually WRONG)
print_args "$*"      # receives 1 arg: "hello world foo"
```

**Rule:** Always use `"$@"` when passing arguments to preserve spacing and quoting.

### Q9. How does arithmetic work in bash?

**Answer:**

Bash variables are strings by default, so you need special syntax for math:

```bash
# Method 1: $(( )) — arithmetic expansion (recommended)
a=10
b=3
echo $((a + b))    # 13
echo $((a - b))    # 7
echo $((a * b))    # 30
echo $((a / b))    # 3  (integer division only)
echo $((a % b))    # 1  (modulo)
echo $((a ** 2))   # 100 (exponent)

# Increment / decrement
count=0
((count++))
((count += 5))
echo $count        # 6

# Method 2: let
let result=a*b
echo $result       # 30

# Method 3: expr (old style, avoid)
result=$(expr $a + $b)

# Method 4: bc — for floating point math
echo "scale=2; 10 / 3" | bc    # 3.33
echo "scale=4; sqrt(2)" | bc   # 1.4142
```

## Input & Output

### Q10. How do you read user input in a shell script?

**Answer:**

```bash
#!/bin/bash

# Basic read
read name
echo "You entered: $name"

# Read with a prompt
read -p "Enter your name: " name
echo "Hello, $name!"

# Read silently (for passwords)
read -sp "Enter password: " password
echo ""   # newline after silent input
echo "Password length: ${#password}"

# Read with a timeout
read -t 10 -p "Answer within 10 seconds: " answer
if [ $? -ne 0 ]; then
    echo "Timed out!"
fi

# Read multiple variables
read -p "Enter first and last name: " first last
echo "First: $first, Last: $last"

# Read into an array
read -a colors -p "Enter colors (space-separated): "
echo "First color: ${colors[0]}"
echo "All colors: ${colors[@]}"
```

### Q11. How do input/output redirection and pipes work?

**Answer:**

**Redirection** sends command output to a file, or reads input from a file.

```bash
# Output redirection
echo "hello" > file.txt       # write (overwrites file)
echo "world" >> file.txt      # append (adds to file)

# Input redirection
sort < names.txt              # sort reads from file instead of keyboard
wc -l < file.txt              # count lines in file

# Redirect stderr (error output)
ls /nonexistent 2> errors.txt         # stderr to file
ls /nonexistent 2>> errors.txt        # append stderr
ls /nonexistent 2>/dev/null           # discard errors

# Redirect both stdout and stderr
./script.sh > output.txt 2>&1         # both to same file
./script.sh &> output.txt             # shorthand (bash only)

# Here document (multi-line input)
cat << EOF
Line 1
Line 2
Line 3
EOF
```

**Pipes** connect stdout of one command to stdin of the next:
```bash
# Basic pipe
ls -la | grep ".txt"
cat file.txt | sort | uniq | wc -l

# Practical examples
ps aux | grep nginx             # find nginx processes
df -h | grep "/dev/sda"        # check disk space
cat /var/log/syslog | tail -100 | grep "error"

# tee: write to file AND keep pipe going
ls -la | tee output.txt | wc -l
```

## Conditionals

### Q12. How do you write if/else statements in bash?

**Answer:**

```bash
#!/bin/bash

# Basic if
if [ condition ]; then
    echo "true"
fi

# if-else
if [ "$name" = "Alice" ]; then
    echo "Hello Alice!"
else
    echo "You are not Alice"
fi

# if-elif-else
score=75

if [ $score -ge 90 ]; then
    echo "Grade: A"
elif [ $score -ge 80 ]; then
    echo "Grade: B"
elif [ $score -ge 70 ]; then
    echo "Grade: C"
else
    echo "Grade: F"
fi
```

**Two bracket styles:**
```bash
# Single brackets [ ] — POSIX compatible
if [ "$a" = "$b" ]; then ...

# Double brackets [[ ]] — bash only, more powerful (preferred)
if [[ "$a" == "$b" ]]; then ...
if [[ "$str" =~ ^[0-9]+$ ]]; then ...   # regex matching
if [[ -z "$var" && "$count" -gt 0 ]]; then ...   # logical operators
```

### Q13. What are the common test operators in bash?

**Answer:**

**String comparisons:**
```bash
[ "$a" = "$b" ]      # equal (use = not == in [ ])
[ "$a" != "$b" ]     # not equal
[ -z "$a" ]          # true if string is empty
[ -n "$a" ]          # true if string is NOT empty
[[ "$a" == "foo*" ]] # pattern match (double brackets only)
[[ "$a" =~ ^[0-9]+$ ]] # regex match
```

**Integer comparisons:**
```bash
[ "$a" -eq "$b" ]    # equal
[ "$a" -ne "$b" ]    # not equal
[ "$a" -lt "$b" ]    # less than
[ "$a" -le "$b" ]    # less than or equal
[ "$a" -gt "$b" ]    # greater than
[ "$a" -ge "$b" ]    # greater than or equal
```

**File tests:**
```bash
[ -e "$file" ]       # exists (file or directory)
[ -f "$file" ]       # exists and is a regular file
[ -d "$dir" ]        # exists and is a directory
[ -r "$file" ]       # exists and is readable
[ -w "$file" ]       # exists and is writable
[ -x "$file" ]       # exists and is executable
[ -s "$file" ]       # exists and is NOT empty
[ -L "$file" ]       # exists and is a symbolic link
[ "$f1" -nt "$f2" ]  # f1 is newer than f2
[ "$f1" -ot "$f2" ]  # f1 is older than f2
```

**Logical operators:**
```bash
[ cond1 ] && [ cond2 ]        # AND (POSIX)
[ cond1 ] || [ cond2 ]        # OR  (POSIX)
[[ cond1 && cond2 ]]          # AND (bash)
[[ cond1 || cond2 ]]          # OR  (bash)
[ ! condition ]                # NOT
```

### Q14. How does the `case` statement work?

**Answer:**

`case` is like a switch statement — cleaner than multiple `if/elif` when comparing one variable against many values.

```bash
#!/bin/bash

read -p "Enter a fruit: " fruit

case "$fruit" in
    apple)
        echo "You chose apple"
        ;;
    banana | plantain)         # multiple patterns with |
        echo "You chose a banana-type fruit"
        ;;
    orange | "blood orange")
        echo "You chose a citrus fruit"
        ;;
    *)                         # default case (like else)
        echo "Unknown fruit: $fruit"
        ;;
esac
```

**Pattern matching in case:**
```bash
case "$filename" in
    *.txt)   echo "Text file" ;;
    *.jpg | *.png | *.gif)  echo "Image file" ;;
    *.sh)    echo "Shell script" ;;
    *)       echo "Unknown type" ;;
esac
```

## Loops

### Q15. What types of loops are available in bash?

**Answer:**

**for loop — iterate over a list:**
```bash
# List of values
for fruit in apple banana orange; do
    echo "Fruit: $fruit"
done

# C-style for loop
for ((i = 0; i < 5; i++)); do
    echo "i = $i"
done

# Range with brace expansion
for i in {1..10}; do
    echo "Number: $i"
done

# Range with step
for i in {0..20..5}; do    # 0, 5, 10, 15, 20
    echo "$i"
done

# Iterate over files
for file in *.txt; do
    echo "Processing: $file"
done

# Iterate over command output
for user in $(cut -d: -f1 /etc/passwd); do
    echo "User: $user"
done
```

**while loop — run while condition is true:**
```bash
count=1
while [ $count -le 5 ]; do
    echo "Count: $count"
    ((count++))
done

# Read file line by line
while IFS= read -r line; do
    echo "Line: $line"
done < file.txt

# Infinite loop
while true; do
    echo "Running..."
    sleep 1
done
```

**until loop — run until condition becomes true:**
```bash
count=1
until [ $count -gt 5 ]; do
    echo "Count: $count"
    ((count++))
done
```

### Q16. How do `break` and `continue` work in loops?

**Answer:**

```bash
# break — exits the loop entirely
for i in {1..10}; do
    if [ $i -eq 5 ]; then
        break       # stop loop when i reaches 5
    fi
    echo "$i"
done
# Output: 1 2 3 4

# continue — skips the rest of the current iteration
for i in {1..10}; do
    if [ $((i % 2)) -eq 0 ]; then
        continue    # skip even numbers
    fi
    echo "$i"
done
# Output: 1 3 5 7 9

# break with nested loops — break N exits N levels
for i in 1 2 3; do
    for j in a b c; do
        if [ "$j" = "b" ]; then
            break 2    # exit both loops
        fi
        echo "$i-$j"
    done
done
```

## Functions

### Q17. How do you define and call functions in bash?

**Answer:**

```bash
#!/bin/bash

# Method 1: function keyword
function greet() {
    echo "Hello, $1!"
}

# Method 2: without keyword (POSIX compatible, preferred)
greet() {
    echo "Hello, $1!"
}

# Call a function
greet "Alice"      # Hello, Alice!
greet "Bob"        # Hello, Bob!

# Function with multiple parameters
add() {
    local result=$(( $1 + $2 ))
    echo $result
}

sum=$(add 5 3)
echo "Sum: $sum"   # Sum: 8

# Function must be defined BEFORE it is called
say_hi          # ERROR: function not found yet

say_hi() {
    echo "Hi!"
}

say_hi          # works fine
```

### Q18. How do functions return values in bash?

**Answer:**

Bash functions don't return values like other languages — they return **exit codes** (0–255). To return actual data, use `echo` and capture with `$()`.

```bash
#!/bin/bash

# Method 1: echo + command substitution (most common)
get_name() {
    echo "Alice"
}

name=$(get_name)
echo "Name: $name"      # Name: Alice

# Method 2: use a global variable
get_info() {
    RESULT="some data"
}

get_info
echo $RESULT            # some data

# Method 3: exit code for success/failure
is_even() {
    if (( $1 % 2 == 0 )); then
        return 0    # true / success
    else
        return 1    # false / failure
    fi
}

if is_even 4; then
    echo "4 is even"
fi

is_even 7
echo "Exit code: $?"    # 1
```

### Q19. What is the `local` keyword and why is it important?

**Answer:**

By default, variables in bash are **global**. The `local` keyword restricts a variable's scope to the function it's declared in, preventing it from affecting the rest of the script.

```bash
#!/bin/bash

# WITHOUT local — variable leaks out (dangerous)
bad_function() {
    name="inside function"
}

name="global"
bad_function
echo $name        # "inside function" — oops! global was changed

# WITH local — variable is confined to the function
good_function() {
    local name="inside function"
    echo "Inside: $name"
}

name="global"
good_function
echo "Outside: $name"   # "global" — safe!
```

**Best practice:** Always use `local` for variables inside functions unless you intentionally need to modify a global variable.

### Q20. How do you work with files and directories in a script?

```bash
fi
```

## String & Array Operations

### Q22. How do you manipulate strings in bash?

**Answer:**

```bash
#!/bin/bash

str="Hello, World!"

# Length
echo ${#str}                    # 13

# Substring (position, length)
echo ${str:7}                   # World!
echo ${str:7:5}                 # World

# Replace (first occurrence)
echo ${str/World/Bash}          # Hello, Bash!

# Replace (all occurrences)
text="aabbcc"
echo ${text//a/x}               # xxbbcc

# Uppercase / lowercase (bash 4+)
echo ${str^^}                   # HELLO, WORLD!
echo ${str,,}                   # hello, world!
echo ${str^}                    # First char uppercase

# Remove prefix
filename="report_2024.txt"
echo ${filename#report_}        # 2024.txt  (remove shortest prefix)
echo ${filename##*_}            # 2024.txt  (remove longest prefix match)

# Remove suffix
echo ${filename%.txt}           # report_2024  (remove shortest suffix)
echo ${filename%.*}             # report_2024

# Default value if variable is empty
name=""
echo ${name:-"default"}         # "default" (use default if empty)
echo ${name:="Alice"}           # assign "Alice" if empty

# Check if string contains substring
if [[ "$str" == *"World"* ]]; then
    echo "Contains World"
fi

# Split string on delimiter
IFS=',' read -ra parts <<< "a,b,c,d"
echo ${parts[0]}                # a
echo ${parts[@]}                # a b c d
```

### Q23. How do you use arrays in bash?

**Answer:**

```bash
#!/bin/bash

# Declare array
fruits=("apple" "banana" "orange" "grape")
empty_array=()

# Access elements
echo ${fruits[0]}               # apple (0-indexed)
echo ${fruits[2]}               # orange
echo ${fruits[-1]}              # grape (last element, bash 4+)

# All elements
echo ${fruits[@]}               # apple banana orange grape
echo ${fruits[*]}               # same (differs when quoted)

# Number of elements
echo ${#fruits[@]}              # 4

# Array slice (start, count)
echo ${fruits[@]:1:2}          # banana orange

# Add elements
fruits+=("mango")
fruits[5]="cherry"

# Remove an element (leaves a gap)
unset fruits[1]                 # removes "banana"
echo ${fruits[@]}               # apple orange grape mango cherry

# Loop through array
for fruit in "${fruits[@]}"; do
    echo "Fruit: $fruit"
done

# Loop with index
for i in "${!fruits[@]}"; do
    echo "$i: ${fruits[$i]}"
done

# Check if array contains a value
target="orange"
for item in "${fruits[@]}"; do
    if [ "$item" = "$target" ]; then
        echo "Found $target!"
        break
    fi
done
```

## Process & System

### Q24. How do you manage background processes in a script?

**Answer:**

```bash
#!/bin/bash

# Run a command in the background with &
sleep 10 &
echo "Sleep is running in background, PID: $!"

# Wait for a specific background process
sleep 5 &
bg_pid=$!
echo "Waiting for PID $bg_pid..."
wait $bg_pid
echo "Done! Exit code: $?"

# Wait for ALL background processes
job1 &
job2 &
job3 &
wait                    # waits for all background jobs
echo "All jobs complete"

# Check if a process is running
if ps -p $pid > /dev/null 2>&1; then
    echo "Process $pid is running"
fi

# Kill a process
kill $pid              # send SIGTERM (graceful)
kill -9 $pid           # send SIGKILL (force)
kill -0 $pid           # just check if process exists

# Run script and keep it running after terminal closes
nohup ./longscript.sh > output.log 2>&1 &
```

### Q25. How do you handle errors and use `set` options for safer scripts?

**Answer:**

By default, bash continues executing even when a command fails. These `set` options make scripts safer:

```bash
#!/bin/bash
set -e          # exit immediately on any error
set -u          # treat unset variables as errors
set -o pipefail # catch errors in pipes (not just last command)
set -x          # print each command before running (debug mode)

# Combine them (common practice)
set -euo pipefail
```

**What each option does:**

```bash
# Without set -e: script continues after failure
rm /nonexistent
echo "This still runs"    # still executes!

# With set -e: script stops
set -e
rm /nonexistent
echo "This never runs"    # script exits here

# Without set -u: unset variables are empty strings
echo $UNDEFINED_VAR       # prints nothing

# With set -u: unset variables cause an error
set -u
echo $UNDEFINED_VAR       # bash: UNDEFINED_VAR: unbound variable

# Without set -o pipefail: only last command's exit code matters
cat nonexistent.txt | grep "foo"
echo $?    # 0 (grep succeeded) — error hidden!

# With set -o pipefail: pipe fails if ANY command fails
set -o pipefail
cat nonexistent.txt | grep "foo"
echo $?    # 1 (cat failed) — error captured!
```

**Trap — run cleanup on exit or error:**
```bash
#!/bin/bash
set -euo pipefail

# cleanup function runs on exit
cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/tempfile.$$
}

trap cleanup EXIT        # run on any exit
trap cleanup ERR         # run on error
trap "echo Interrupted" INT    # run on Ctrl+C

# Create temp file
touch /tmp/tempfile.$$
echo "Doing work..."
# cleanup() runs automatically when script exits
```

## Common Commands & Tricks

### Q26. What are the most useful text processing commands in shell scripts?

**Answer:**

**`cut` — extract columns:**
```bash
echo "Alice:30:Engineer" | cut -d: -f1     # Alice
echo "Alice:30:Engineer" | cut -d: -f2,3   # 30:Engineer
cut -d, -f1 data.csv                        # first column of CSV
```

**`awk` — powerful field processing:**
```bash
# Print specific fields
echo "Alice 30 Engineer" | awk '{print $1, $3}'    # Alice Engineer

# With delimiter
awk -F: '{print $1}' /etc/passwd                    # list all usernames

# Filter and print
awk '$3 > 1000 {print $1}' /etc/passwd             # UIDs above 1000

# Sum a column
awk '{sum += $1} END {print sum}' numbers.txt
```

**`sed` — stream editor:**
```bash
# Replace text
sed 's/old/new/' file.txt         # replace first occurrence per line
sed 's/old/new/g' file.txt        # replace all occurrences
sed -i 's/old/new/g' file.txt     # edit file in-place

# Delete lines
sed '/pattern/d' file.txt         # delete matching lines
sed '5d' file.txt                 # delete line 5
sed '5,10d' file.txt              # delete lines 5-10

# Print specific lines
sed -n '5,10p' file.txt           # print lines 5–10
```

**`sort` and `uniq`:**
```bash
sort file.txt                      # sort alphabetically
sort -n numbers.txt                # sort numerically
sort -r file.txt                   # reverse sort
sort -k2 data.txt                  # sort by 2nd field
sort file.txt | uniq               # remove duplicate lines
sort file.txt | uniq -c            # count occurrences
sort file.txt | uniq -d            # show only duplicates
```

**`xargs` — build commands from input:**
```bash
find . -name "*.log" | xargs rm           # delete all .log files
find . -name "*.txt" | xargs wc -l        # count lines in all txt files
echo "file1 file2 file3" | xargs -n1 echo # one argument per line
```

### Q27. How do you write a script that accepts command-line arguments and flags?

**Answer:**

```bash
#!/bin/bash
# Usage: ./deploy.sh -e production -v 1.2.3

# Method 1: Simple positional arguments
# ./script.sh arg1 arg2
echo "First arg: $1"
echo "Second arg: $2"

# Method 2: getopts (built-in flag parsing)
usage() {
    echo "Usage: $0 -e <environment> -v <version> [-d]"
    exit 1
}

debug=false
environment=""
version=""

while getopts "e:v:dh" opt; do
    case $opt in
        e) environment="$OPTARG" ;;
        v) version="$OPTARG" ;;
        d) debug=true ;;
        h) usage ;;
        *) usage ;;
    esac
done

# Validate required arguments
if [ -z "$environment" ] || [ -z "$version" ]; then
    echo "Error: environment and version are required"
    usage
fi

echo "Deploying version $version to $environment"
if $debug; then
    echo "Debug mode enabled"
fi
```

**Running:**
```bash
./deploy.sh -e production -v 1.2.3
./deploy.sh -e staging -v 2.0.0 -d
./deploy.sh -h
```

### Q28. How do you schedule a shell script with cron?

**Answer:**

**Cron** is a time-based job scheduler. Edit the cron table with `crontab -e`.

**Cron syntax:**
```
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of week (0=Sun, 6=Sat)
│ │ │ └──── Month (1-12)
│ │ └────── Day of month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
```

**Common examples:**
```bash
# Every minute
* * * * * /home/user/script.sh

# Every day at 2:30 AM
30 2 * * * /home/user/backup.sh

# Every Monday at 9 AM
0 9 * * 1 /home/user/report.sh

# Every 15 minutes
*/15 * * * * /home/user/check.sh

# First day of every month at midnight
0 0 1 * * /home/user/monthly.sh

# Weekdays at 8 AM
0 8 * * 1-5 /home/user/workday.sh

# Log output from cron job
30 2 * * * /home/user/backup.sh >> /var/log/backup.log 2>&1
```

**Manage crontab:**
```bash
crontab -e      # edit your crontab
crontab -l      # list your crontab
crontab -r      # remove your crontab
crontab -u user -l   # list another user's crontab (as root)
```
