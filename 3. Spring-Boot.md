## 1. What is Spring Boot, and how does it differ from Spring Framework?

**Answer:**

Spring Boot is an opinionated framework built on top of Spring Framework that simplifies application development and deployment.

**Key Features**

- Auto-configuration: configures beans based on classpath and existing beans
- Embedded servers: Tomcat/Jetty/Undertow â€” no WAR deployment needed
- Starter dependencies: pre-configured dependency bundles
- Production-ready features: health checks, metrics, externalized configuration
- No XML configuration: Java-based annotations

Bottom line: Spring Framework provides core modules (IoC, AOP, MVC, Data); Spring Boot provides tooling and defaults to use them with minimal configuration.

## 2. What are Spring Boot starters?

**Answer:**

Starters are pre-packaged dependency bundles that include compatible libraries for a specific functionality.

Example Maven starter:

```xml
<!-- One starter replaces many individual dependencies -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

Common starters:

- `spring-boot-starter-web` â€” REST APIs & Spring MVC (includes Jackson, Tomcat)
- `spring-boot-starter-data-jpa` â€” Database access (Hibernate, Spring Data JPA, HikariCP)
- `spring-boot-starter-security` â€” Spring Security
- `spring-boot-starter-test` â€” JUnit 5, Mockito, AssertJ

Benefits: no version conflicts, faster setup, auto-configuration triggers.

## 3. What is Spring Boot auto-configuration?

**Answer:**

Auto-configuration automatically configures beans based on classpath, existing beans, and properties.

Example â€” DataSource auto-configuration:

```properties
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.driver-class-name=org.h2.Driver
```

Spring Boot will auto-create a `DataSource` bean, configure HikariCP, and provide `JdbcTemplate` if available.

Key conditional annotations:

- `@ConditionalOnClass`
- `@ConditionalOnMissingBean`
- `@ConditionalOnProperty`

Customization: override by defining your own bean, exclude auto-configs via `@SpringBootApplication(exclude = ...)`, or enable `debug=true` to inspect auto-configuration.

## 4. How do you create a REST API using Spring Boot?

**Answer (summary):**

1. Add `spring-boot-starter-web` dependency.
2. Create a `@RestController` with `@RequestMapping` and handler methods (`@GetMapping`, `@PostMapping`, ...).
3. Run application with `@SpringBootApplication` and `SpringApplication.run(...)`.

Example controller:

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    private final UserService userService;

    public UserController(UserService userService) { this.userService = userService; }

    @GetMapping
    public List<User> getAllUsers() { return userService.findAll(); }

    @GetMapping("/{id}")
    public ResponseEntity<User> getUser(@PathVariable Long id) { /* ... */ }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public User createUser(@Valid @RequestBody UserDTO dto) { return userService.create(dto); }

    @PutMapping("/{id}")
    public User updateUser(@PathVariable Long id, @Valid @RequestBody UserDTO dto) { return userService.update(id, dto); }

    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    public void deleteUser(@PathVariable Long id) { userService.delete(id); }
}
```

Result: API available at `http://localhost:8080/api/users` with automatic JSON serialization.

## 5. What is the role of `application.properties` or `application.yml`?

**Answer:**

Configuration files externalize settings allowing environment-specific behavior without code changes.

Properties example:

```properties
spring.datasource.url=jdbc:mysql://localhost/db
spring.datasource.username=root
spring.datasource.password=${DB_PASSWORD}
```

YAML example:

```yaml
server:
  port: 8081
spring:
  datasource:
    url: jdbc:mysql://localhost/db
    username: root
    password: ${DB_PASSWORD}
```

Profiles: `application-dev.yml`, `application-prod.yml`, activated with `spring.profiles.active`.

Binding properties:

```java
@Value("${app.feature.enabled:false}")
private boolean featureEnabled;

@ConfigurationProperties(prefix = "app.mail")
public record MailProperties(String host, int port, String from) {}
```

## 6. What is the difference between `@SpringBootApplication` and `@EnableAutoConfiguration`?

**Answer:**

`@SpringBootApplication` is a convenience annotation that combines:

- `@SpringBootConfiguration`
- `@EnableAutoConfiguration`
- `@ComponentScan`

Use `@SpringBootApplication` on the main class; use individual annotations only when explicit control is needed.

## 7. How do you secure a Spring Boot application?

**Answer:**

1. Add `spring-boot-starter-security` dependency.
2. Configure security using `SecurityFilterChain` beans (Spring Security 6+).
3. Use `PasswordEncoder` such as `BCryptPasswordEncoder`.

Example security config (outline):

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            // configure auth, csrf, session, etc.
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() { return new BCryptPasswordEncoder(); }
}
```

Authentication options: in-memory, JDBC, LDAP, OAuth2/OpenID Connect, JWT, custom providers.

Method-level security example:

```java
@EnableMethodSecurity
@Service
public class UserService {
    @PreAuthorize("hasRole('ADMIN')")
    public void deleteUser(Long id) { ... }
}
```

Security checklist: use BCrypt, enable HTTPS in production, secure actuator endpoints, store secrets securely, validate inputs, implement rate limiting.

## 8. What is Spring Boot Actuator, and why is it useful?

**Answer:**

Actuator provides production-ready monitoring & management endpoints (HTTP or JMX).

Add dependency: `spring-boot-starter-actuator`.

Use cases: liveness/readiness probes, metrics export, thread dumps, bean inspection, dynamic log-level changes.

Secure sensitive actuator endpoints in production.

## 9. How do you handle exceptions in Spring Boot REST APIs?

**Answer:**

Recommended approach: `@RestControllerAdvice` global exception handler.

Example:

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(ResourceNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ErrorResponse handleNotFound(ResourceNotFoundException ex) { /* ... */ }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ErrorResponse handleValidation(MethodArgumentNotValidException ex) { /* ... */ }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ErrorResponse handleAll(Exception ex) { /* ... */ }
}
```

Error response DTO (Problem Details style):

```java
public record ErrorResponse(
    int status,
    String code,
    String message,
    LocalDateTime timestamp,
    String path,
    List<String> details
) { /* constructors */ }
```

Best practices: consistent error format, log stack traces server-side, return safe client messages, use proper HTTP status codes, include correlation IDs, never expose internal details.
