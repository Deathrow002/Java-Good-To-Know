Java and Spring Interview Questions and Answers
================================================

================================================================================
CORE JAVA (FUNDAMENTALS & OOP)
================================================================================

1. What are the main features of Java?

Answer:
Java is:
- Object-Oriented: Everything is treated as an object
- Platform Independent: Bytecode runs on any OS with JVM
- Robust: Strong memory management and exception handling
- Secure: No explicit pointer usage, supports security APIs
- Multithreaded: Built-in support for concurrent execution
- High Performance: Just-In-Time compiler improves speed
- Distributed: Supports networking and remote method invocation

--------------------------------------------------------------------------------

2. Explain the concept of Object-Oriented Programming in Java

Answer:
OOP in Java is based on four pillars:
- Encapsulation: Wrapping data and methods into a single unit (class)
- Inheritance: Reusing code via parent-child relationships
- Polymorphism: One interface, many implementations (method overloading/overriding)
- Abstraction: Hiding internal details and exposing only functionality

--------------------------------------------------------------------------------

3. What is the difference between == and .equals() in Java?

Answer:
- == compares object references (memory addresses)
- .equals() compares object content (logical equality)

For example, two different String objects with the same value return true with .equals() but false with ==.

--------------------------------------------------------------------------------

4. What are hashCode() and equals() methods used for?

Answer:
- equals() defines logical equality; hashCode() returns an int used by hash-based collections
- The contract: if two objects are equal per equals(), they must return the same hashCode(); equal hashCodes do not guarantee equals()
- Override both together to use objects as keys in HashMap/entries in HashSet

--------------------------------------------------------------------------------

5. Explain method overloading vs method overriding

Answer:
- Overloading: Same method name, different parameters, within the same class
- Overriding: Subclass provides a specific implementation of a method declared in the parent class

Overriding requires same method signature and allows runtime polymorphism.

--------------------------------------------------------------------------------

6. What is the difference between an abstract class and an interface?

Answer:
- Abstract classes can have state, constructors, concrete and abstract methods, and are single-inherited; use them to share common implementation
- Interfaces declare a contract (methods) and from Java 8 can include default/static/private methods; a class can implement multiple interfaces; use interfaces to define capabilities across unrelated classes

--------------------------------------------------------------------------------

7. What are access modifiers in Java?

Answer:
- Top-level classes can be public or package-private (no modifier)
- Member-level modifiers:
  - private (class-only)
  - default/package-private (same package)
  - protected (same package + subclasses)
  - public (visible everywhere)

Prefer the most restrictive sensible modifier, typically private for fields.

--------------------------------------------------------------------------------

8. What is the difference between final, finally, and finalize()?

Answer:
- final is a keyword that prevents reassignment (variables), overriding (methods), or inheritance (classes)
- finally is an exception-handling block that always executes after try/catch
- finalize() is an Object method invoked by the garbage collector for cleanup but is deprecated and not reliable for resource management

--------------------------------------------------------------------------------

9. Explain the concept of immutability in Java

Answer:
An immutable object's state cannot change after construction.

Achieve it by:
- Making fields private and final
- Not exposing setters
- Defensively copying mutable components
- Avoiding methods that mutate internal state

String is the canonical example.

--------------------------------------------------------------------------------

10. What is the difference between String, StringBuilder, and StringBuffer?

Answer:
- String is immutable; modifications produce new objects so it's safe for sharing but expensive for repeated changes
- StringBuilder is mutable and unsynchronized; use it for efficient single-threaded string construction
- StringBuffer is mutable and synchronized; use it when multiple threads will modify the same sequence

================================================================================
JAVA ADVANCED CONCEPTS
================================================================================

1. What is the difference between checked and unchecked exceptions?

Answer:
- Checked exceptions are checked at compile time and must be declared or handled (extends Exception but not RuntimeException). Use them for recoverable, external conditions (IO, SQL)
- Unchecked exceptions are runtime exceptions (extend RuntimeException or Error), not required to be declared or caught, and typically indicate programmer errors (NullPointerException, IllegalArgumentException)

--------------------------------------------------------------------------------

2. How does garbage collection work in Java?

Answer:
The JVM automatically reclaims objects that are no longer reachable from live references.

- Heaps are usually generational (Young, Old/Tenured, sometimes a Permanent/Metaspace area)
- GC algorithms mark reachable objects, sweep or compact unreachable ones, and promote survivors between generations
- Different collectors (serial, parallel, CMS, G1, ZGC, Shenandoah) trade off throughput, latency, and footprint
- You can request GC with System.gc() but the JVM decides timing; finalize() is deprecated and not reliable for cleanup

--------------------------------------------------------------------------------

3. What are Java annotations, and how are they used?

Answer:
Annotations are metadata that attach information to program elements using @ syntax (e.g., @Override, @Deprecated, @Retention).

They guide:
- The compiler
- Tooling
- Runtime frameworks

Common uses:
- Compile-time checks (@Override)
- Suppress warnings (@SuppressWarnings)
- Runtime processing (dependency injection, serialization)
- Code generation

You can create custom annotations and control their retention (SOURCE, CLASS, RUNTIME) and targets (TYPE, METHOD, FIELD, etc.).

--------------------------------------------------------------------------------

4. Explain the concept of reflection in Java

Answer:
Reflection is the ability to inspect and manipulate classes, methods, fields, constructors, and annotations at runtime via java.lang.Class and java.lang.reflect.

Use cases:
- Dependency injection
- Frameworks
- Serialization
- Testing
- Dynamic instantiation

Trade-offs:
- Reflection bypasses compile-time checks
- Has performance overhead
- Can break encapsulation and future compatibility
- Use it judiciously

--------------------------------------------------------------------------------

5. What is the difference between ArrayList and LinkedList?

Answer:
- ArrayList uses a dynamic array backing store — O(1) random access, amortized O(1) append, O(n) insert/delete at arbitrary positions because of shifting
- LinkedList is a doubly-linked list — O(1) insert/delete given a node or at ends, but O(n) random access
- Memory: LinkedList uses more per-element overhead

Choose:
- ArrayList for frequent random reads and iteration
- LinkedList for many insertions/removals at list ends or when using ListIterator for local modifications

--------------------------------------------------------------------------------

6. What is the difference between HashMap, TreeMap, and LinkedHashMap?

Answer:
- HashMap: Unordered, average O(1) put/get using hashing; allows one null key; iteration order is unpredictable
- LinkedHashMap: Hash-based with predictable insertion-order or access-order iteration (maintains a doubly-linked list of entries); similar O(1) performance with extra order guarantees
- TreeMap: Sorted map (natural order or Comparator), implemented as a red-black tree; O(log n) put/get; does not allow null keys if using natural ordering

Use:
- HashMap for speed
- LinkedHashMap when order matters
- TreeMap when you need sorted keys and range queries

--------------------------------------------------------------------------------

7. What are generics in Java, and why are they useful?

Answer:
Generics provide parameterized types (e.g., List<String>) enabling compile-time type safety and eliminating many casts.

Benefits:
- Stronger compile-time checks
- Clearer APIs
- Reusable algorithms

Remember type erasure: generic type parameters are removed at runtime, so you cannot use primitives as type arguments and reflective checks must account for erasure.

Use bounded type parameters (extends/super) to express constraints.

--------------------------------------------------------------------------------

8. Explain the concept of multithreading in Java

Answer:
Multithreading allows concurrent execution of multiple threads within a process.

Thread creation:
- Extending Thread
- Implementing Runnable
- Submitting Callables/Runnables to ExecutorService (preferred)

Important points:
- Threads share heap memory and must coordinate access to shared mutable state to avoid race conditions
- Thread lifecycle: New → Runnable → Running → Blocked/Waiting → Terminated
- Use java.util.concurrent constructs (locks, atomics, executors, concurrent collections) rather than low-level primitives where possible

--------------------------------------------------------------------------------

9. What is the difference between synchronized methods and blocks?

Answer:
- Synchronized method locks the entire method on the instance (or Class object for static methods)
- Synchronized block locks only the specified object and scopes the critical section narrowly

Benefits:
- Synchronized blocks reduce contention and allow finer-grained locking (better performance and concurrency)
- Methods are simpler but coarser

Prefer minimal scope locks and prefer explicit lock objects for clarity when needed.

--------------------------------------------------------------------------------

10. What are Java Streams, and how do they differ from Collections?

Answer:
Streams (java.util.stream) provide a pipeline-based, functional-style API to process sequences of elements (map, filter, reduce).

Key differences:
- Data storage: Streams do not store data; Collections do
- Iteration: Streams support lazy, possibly parallel, internal iteration and are single-use (consumable); Collections are reusable and support external iteration and modification
- Purpose: Streams focus on data processing (declarative transformations) without mutating the source (functional style)

Use:
- Streams for concise, composable transformations and parallel processing
- Collections to store and mutate data

--------------------------------------------------------------------------------

11. What is the Java Collections Framework?

Answer:
The Java Collections Framework is a unified architecture for representing and manipulating collections of objects.

Core interfaces:
- Collection: root interface for single-element containers
- List: ordered collection allowing duplicates (ArrayList, LinkedList)
- Set: unordered collection with no duplicates (HashSet, TreeSet, LinkedHashSet)
- Queue/Deque: collections for holding elements prior to processing (PriorityQueue, ArrayDeque)
- Map: key-value pairs (HashMap, TreeMap, LinkedHashMap) — not a Collection subtype

Key characteristics:
- Provides standard algorithms (sort, shuffle, reverse, binarySearch) via Collections utility class
- Supports fail-fast iterators that throw ConcurrentModificationException on structural modification during iteration
- Concurrent collections (ConcurrentHashMap, CopyOnWriteArrayList) are thread-safe alternatives

Best practices:
- Program to interfaces, not implementations (List<String> instead of ArrayList<String>)
- Choose collections based on access patterns, ordering needs, and thread safety requirements
- Use Collections.unmodifiableList/Set/Map for immutable views

--------------------------------------------------------------------------------

12. What are common Stream API operations and best practices?

Answer:
Stream operations are divided into intermediate (lazy, return Stream) and terminal (eager, produce result).

Intermediate operations:
- filter(): select elements matching a predicate
- map(): transform elements
- flatMap(): flatten nested structures
- sorted(): sort elements
- distinct(): remove duplicates
- limit()/skip(): truncate or skip elements
- peek(): debug/inspect without consuming

Terminal operations:
- collect(): accumulate into collection (Collectors.toList(), toSet(), groupingBy(), joining())
- forEach(): iterate and perform action
- reduce(): combine elements into single result
- count(), min(), max(), findFirst(), findAny(): aggregate operations
- anyMatch(), allMatch(), noneMatch(): boolean checks

Best practices:
- Prefer method references over lambdas when possible
- Avoid side effects in stream operations
- Use parallel streams only for CPU-intensive operations on large datasets
- Close streams from I/O sources (Files.lines()) using try-with-resources
- Consider Optional for findFirst()/findAny() to handle empty results gracefully

================================================================================
SPRING CORE
================================================================================

1. What is the Spring Framework, and why is it used?

Answer:
Spring is an open-source Java application framework that provides a lightweight, modular programming model centered on:
- Core IoC container
- Dependency Injection
- AOP
- Data access
- Transaction management
- Web frameworks

It helps build robust, testable, and maintainable enterprise applications.

--------------------------------------------------------------------------------

2. Explain the concept of Inversion of Control (IoC)

Answer:
IoC is a design principle where control of object creation and wiring is inverted from application code to a container.

Objects declare dependencies and the container instantiates and injects them, enabling loose coupling and easier testing.

--------------------------------------------------------------------------------

3. What is Dependency Injection (DI) in Spring?

Answer:
DI is the concrete pattern used to implement IoC in Spring: the container supplies required dependencies to beans via constructors, factory methods, or property setters so classes do not create or look up their collaborators themselves.

--------------------------------------------------------------------------------

4. What are the different types of dependency injection in Spring?

Answer:
Main types are:
- Constructor injection
- Setter (property) injection
- Field injection (via annotations)

Constructor injection is preferred for mandatory dependencies and immutability; setter/field injection can be used for optional or framework-managed wiring.

--------------------------------------------------------------------------------

5. What is the difference between @Component, @Service, @Repository, and @Controller?

Answer:
They are stereotype annotations (all meta-annotated with @Component) used for component scanning and layer semantics:
- @Component is generic
- @Service marks service-layer beans
- @Repository marks persistence/DAO beans and enables exception translation
- @Controller designates web MVC controllers

--------------------------------------------------------------------------------

6. What is the difference between BeanFactory and ApplicationContext?

Answer:
ApplicationContext extends BeanFactory and adds enterprise features:
- Eager startup of singletons
- Message resource handling
- Event propagation
- Automatic registration of BeanPostProcessors
- Convenience utilities

BeanFactory is a lightweight, lazy-loading container used when minimal footprint is required.

--------------------------------------------------------------------------------

7. What are Spring bean scopes?

Answer:
Standard scopes include:
- singleton (one instance per container, default)
- prototype (new instance per request)
- Web-aware scopes: request, session, application, and websocket

Custom scopes are also possible.

--------------------------------------------------------------------------------

8. How does Spring handle bean lifecycle?

Answer:
The container manages instantiation, dependency injection, and lifecycle callbacks.

Hooks include:
- @PostConstruct/@PreDestroy
- InitializingBean/DisposableBean interfaces
- Custom init-method/destroy-method configuration
- BeanPostProcessor extension points for custom behavior

--------------------------------------------------------------------------------

9. What is the role of @Autowired annotation?

Answer:
@Autowired instructs Spring to resolve and inject bean dependencies by type into constructors, setters, fields, or methods.

It supports:
- Optional injection
- Qualifiers for disambiguation
- Integration with component scanning and the ApplicationContext

--------------------------------------------------------------------------------

10. What is the difference between constructor injection and setter injection?

Answer:
- Constructor injection provides dependencies at creation time, supports immutability, and prevents circular dependencies
- Setter injection allows optional or reconfigurable dependencies post-construction and can be more flexible but does not enforce mandatory wiring at instantiation

================================================================================
SPRING BOOT
================================================================================

1. What is Spring Boot, and how does it differ from Spring Framework?

Answer:
Spring Boot is an opinionated, production-ready layer built on top of the Spring Framework that removes boilerplate by providing:
- Auto-configuration
- Embedded servers
- Sensible defaults
- Starters

Spring Framework is the underlying modular framework for dependency injection, AOP, data access, web MVC, etc.

Spring Boot streamlines setup and runtime packaging so you can run self-contained applications quickly.

--------------------------------------------------------------------------------

2. What are Spring Boot starters?

Answer:
Starters are curated dependency descriptors (starter POMs/Gradle modules) that pull in a set of related libraries and sensible versions for a given feature (e.g., web, data-jpa, security, actuator).

They simplify dependency management and accelerate project setup.

--------------------------------------------------------------------------------

3. What is Spring Boot auto-configuration?

Answer:
Auto-configuration inspects the application classpath and existing beans and conditionally registers beans to provide sensible defaults (via @Conditional annotations).

It is non-invasive: custom user beans override auto-configured ones and you can exclude specific auto-configurations.

--------------------------------------------------------------------------------

4. How do you create a REST API using Spring Boot?

Answer:
1. Add the web starter
2. Create @RestController classes with mapping annotations
3. Implement services/repositories
4. Run the app using SpringApplication.run()

--------------------------------------------------------------------------------

5. What is the role of application.properties or application.yml?

Answer:
These files externalize configuration:
- Server port
- Datasource credentials
- Logging
- Custom feature toggles
- Profile settings
- Environment-specific properties

YAML supports hierarchical structure and lists; properties files are flat key-value. Spring resolves properties from many sources with a defined precedence.

--------------------------------------------------------------------------------

6. What is the difference between @SpringBootApplication and @EnableAutoConfiguration?

Answer:
- @SpringBootApplication is a composite annotation that includes @SpringBootConfiguration, @EnableAutoConfiguration, and @ComponentScan
- @EnableAutoConfiguration only enables the auto-configuration feature

Use the composite for the common application bootstrap.

--------------------------------------------------------------------------------

7. How do you secure a Spring Boot application?

Answer:
1. Add spring-boot-starter-security
2. Configure authentication/authorization via a SecurityFilterChain bean (modern Spring Security 5.7+)
3. Use PasswordEncoder (BCrypt)
4. Integrate JWT/OAuth2 for stateless APIs
5. Secure actuator endpoints
6. Enforce HTTPS
7. Apply role-based access with @PreAuthorize
8. Avoid exposing secrets in config files

--------------------------------------------------------------------------------

8. What is Spring Boot Actuator, and why is it useful?

Answer:
Actuator provides production-ready operational endpoints:
- Health
- Metrics
- Info
- Environment
- Beans
- Tracing

It enables:
- Health checks
- Metrics export
- Diagnostics
- Integration with Prometheus/Grafana and APM tools

Can be extended or restricted as needed.

--------------------------------------------------------------------------------

9. How do you handle exceptions in Spring Boot REST APIs?

Answer:
Use @ControllerAdvice (or @RestControllerAdvice) with @ExceptionHandler methods to centralize error handling and return structured error responses.

Best practices:
- Optionally extend ResponseEntityExceptionHandler for common Spring exceptions
- Provide meaningful HTTP status codes, error codes, and a consistent error DTO (problem details / RFC 7807)

================================================================================
SPRING SECURITY & DATA
================================================================================

1. What is Spring Security, and how does it work?

Answer:
Spring Security provides authentication, authorization, and protection against common web attacks. It's implemented as an ordered SecurityFilterChain that processes every request.

Key building blocks:
- SecurityFilterChain: ordered filters for authn (Basic, Form, Bearer), headers, CSRF, and authorization.
- AuthenticationManager/Providers: verify credentials via UserDetailsService, LDAP, OAuth2/JWT, custom providers.
- SecurityContext: stores the current Authentication (principal and authorities).
- Authorization: URL rules and method security using @EnableMethodSecurity with @PreAuthorize/@PostAuthorize.

Typical flow:
- A filter extracts credentials and creates an Authentication token.
- AuthenticationManager delegates to providers; on success, the context is populated.
- Authorization checks are applied before the controller or at method boundaries.

Modern Spring Security favors bean-based configuration with the HttpSecurity DSL and stateless APIs for microservices.

--------------------------------------------------------------------------------

2. How do you implement authentication and authorization in Spring Security?

Answer:
Configure a SecurityFilterChain and supporting beans for identity and access control.

Authentication options:
- Username/password via UserDetailsService + PasswordEncoder (BCrypt).
- OAuth2/OIDC login with external IdPs via spring-boot-starter-oauth2-client.
- JWT/OAuth2 Resource Server for APIs validating bearer tokens via JWKs.
- LDAP/AD integration for enterprise directories.

Authorization:
- URL rules with authorizeHttpRequests in the filter chain.
- Method security via @EnableMethodSecurity and @PreAuthorize/@PostAuthorize.
- Prefer fine-grained authorities; map roles to authorities for coarse control.

For stateless APIs set SessionCreationPolicy.STATELESS, disable CSRF, and validate tokens per request.

--------------------------------------------------------------------------------

3. What is the difference between JWT and OAuth2 in Spring Security?

Answer:
- JWT: a signed token format carrying claims; enables stateless validation with no server-side session.
- OAuth2: a framework for delegated authorization with standardized flows; access tokens may be JWT or opaque.

When to use:
- Use JWT directly for simple internal services you issue/validate yourself.
- Use OAuth2/OIDC for SSO, refresh tokens, scopes/consent, and third-party IdPs.

Spring support: resource servers via spring-boot-starter-oauth2-resource-server; OAuth2 clients/login via spring-boot-starter-oauth2-client.

--------------------------------------------------------------------------------

4. How do you secure REST APIs in Spring Boot?

Answer:
1. Use stateless auth (JWT or OAuth2 resource server) and set SessionCreationPolicy.STATELESS.
2. Disable CSRF for non-browser token clients; enforce HTTPS/HSTS and secure headers.
3. Configure a SecurityFilterChain that validates bearer tokens.
4. Define least-privilege URL/method rules; add method security for business rules.
5. Validate inputs and standardize errors (Problem Details); rate-limit at the edge.
6. Secure Actuator; externalize secrets to env/vault; avoid leaking sensitive errors.

--------------------------------------------------------------------------------

5. What is CSRF protection in Spring Security?

Answer:
CSRF protects state-changing requests in browser-based apps by requiring a secret token browsers can't forge.

Guidance:
- Enabled by default for session/cookie-based apps; include and validate the token in forms or headers.
- Disable for stateless token APIs (no cookies); rely on authn, CORS, and secure headers instead.
- For SPAs needing JS access to the token, use CookieCsrfTokenRepository.

--------------------------------------------------------------------------------

6. What is Spring Data JPA, and how does it simplify database access?

Answer:
Spring Data JPA builds on JPA to provide repository abstractions that auto-generate implementations for common persistence operations from simple interfaces.

It reduces boilerplate by offering:
- CrudRepository/JpaRepository
- Query derivation from method names
- Pagination/sorting helpers
- Custom query annotations
- Integration with Spring transactions and configuration

--------------------------------------------------------------------------------

7. What is the difference between CrudRepository, JpaRepository, and PagingAndSortingRepository?

Answer:
- CrudRepository provides basic CRUD methods (save, findById, findAll, delete)
- PagingAndSortingRepository extends CrudRepository and adds pagination and sorting methods (findAll(Pageable), findAll(Sort))
- JpaRepository extends PagingAndSortingRepository and adds JPA-specific operations (flush, saveAllAndFlush, batch deletes) and richer JPA semantics

Choose the smallest interface that meets your needs.

--------------------------------------------------------------------------------

8. How do you implement caching in Spring Boot?

Answer:
1. Enable caching with @EnableCaching (or rely on auto-configuration)
2. Provide a CacheManager (Caffeine, EhCache, Redis, or default ConcurrentMapCache)
3. Annotate methods with @Cacheable, @CachePut, and @CacheEvict to control cache population and invalidation
4. Configure cache names, TTLs, and serialization in provider-specific configs
5. Ensure cache eviction on updates to maintain consistency

--------------------------------------------------------------------------------

9. How do you configure transactions in Spring?

Answer:
Use Spring's declarative transaction management:
1. Annotate service-layer methods with @Transactional
2. Configure a PlatformTransactionManager (JpaTransactionManager for JPA)
3. Control propagation, isolation, readOnly, and rollback behavior via annotation attributes

Prefer placing @Transactional on boundary/service methods to keep transaction scope clear and use programmatic transactions only when necessary.

--------------------------------------------------------------------------------

10. What is the difference between lazy loading and eager loading in JPA?

Answer:
- Lazy loading defers loading related entities/collections until they are accessed (proxy or additional query), reducing initial fetch cost but risking LazyInitializationException if session is closed
- Eager loading fetches associations immediately with the parent (join or extra selects), ensuring availability but potentially loading unnecessary data

Best practice:
- Choose lazy by default
- Optimize queries with fetch joins, entity graphs, or DTO projections when you need specific data in a single query
